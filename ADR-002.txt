ADR-002: IMPLEMENTAR LOGGING ESTRUTURADO COM CORRELATION IDs

Status: ACEITO
Data: 19/04/2026
Autor: Equipe de Desenvolvimento

---

1. CONTEXTO

Requisição passa por Order Service → Product Service. Se erro 500, como saber qual serviço causou?

Logs estão separados. Correlation ID único viaja por todos serviços para correlacionar.


2. ALTERNATIVAS CONSIDERADAS

ALTERNATIVA A: Logging estruturado com Correlation ID - ESCOLHIDA

Como funciona:
- Gera UUID único por requisição
- Propaga via header X-Correlation-ID
- Logs JSON incluem correlationId

Exemplo log:
{
  "service": "order-service",
  "correlationId": "abc123",
  "method": "POST",
  "url": "/orders"
}

Vantagens:
✓ Rastreabilidade completa
✓ JSON estruturado
✓ Sem ferramentas extras
✓ Escalável para ELK/Datadog

Desvantagens:
- Logs stdout (perdem se container morre)
- Disciplina para sempre enviar header


ALTERNATIVA B: Logging centralizado custom

Criar microsserviço para logs centralizados.

Vantagens:
+ Logs centralizados desde início
+ Dashboard visual

Desvantagens:
- Mais complexidade
- Overkill para projeto educacional

NÃO escolhido.


ALTERNATIVA C: Logging simples (console.log plain text)

Apenas console.log sem estrutura.

Vantagens:
+ Fácil

Desvantagens:
- Impossível correlacionar
- Sem contexto

NÃO recomendado.
- Sem contexto: timestamp, service, method, etc

NÃO recomendado: não oferece rastreabilidade.


ALTERNATIVA D: Biblioteca logging (Winston, Pino)

Usar Winston para logging avançado.

Vantagens:
+ Profissional
+ Múltiplos outputs

Desvantagens:
- Mais dependências
- Curva aprendizado

NÃO escolhido: console.log + JSON suficiente.


3. DECISÃO

USAR LOGGING ESTRUTURADO COM CORRELATION IDs

Justificativa:
Padrão indústria. Simples, escalável. Aprender Correlation IDs agora economiza tempo depois.


4. CONSEQUÊNCIAS

POSITIVAS:
✓ Rastreabilidade requisições
✓ JSON indexável
✓ Sem dependências extras
✓ Aprendizado valioso

NEGATIVAS:
✗ Logs stdout (não persistentes)
✗ Disciplina para propagar ID


5. IMPLEMENTAÇÃO

Middleware Express:

app.use((req, res, next) => {
    req.correlationId = req.headers['x-correlation-id'] || uuidv4();
    res.set('x-correlation-id', req.correlationId);
    
    console.log(JSON.stringify({
        service: "order-service",
        correlationId: req.correlationId,
        method: req.method,
        url: req.url
    }));
    
    next();
});

Ao chamar serviço:
axios.get('http://product-service:3001/products', {
    headers: { 'x-correlation-id': req.correlationId }
});
```


6. VALIDAÇÃO E TESTES

Testes realizados:
✓ Correlation ID gerado se não fornecido
✓ Propagado entre serviços
✓ Logs JSON válidos
✓ Header retornado

Teste manual:
curl -H "X-Correlation-ID: meu-id-teste" http://localhost:3002/orders
Procurar logs com "meu-id-teste" em ambos serviços.
