ADR-002: IMPLEMENTAR LOGGING ESTRUTURADO COM CORRELATION IDS

Status: ACEITO
Data: 24/04/2026
Autor: Equipe de Desenvolvimento

1. Contexto

O fluxo de requisicoes abrange varios servicos: UI Service, Demand Service, Deadline Checker Service e Email Service.
Quando um erro acontece, precisamos saber qual servico participou daquela operacao.

2. Alternativas consideradas

Alternativa A: logging estruturado com Correlation ID - escolhido
- Gera um UUID unico por requisicao.
- Propaga o valor no header X-Correlation-ID.
- Registra logs em JSON com correlationId, service, method, url e timestamp.

Vantagens:
- Facil de rastrear o fluxo entre servicos.
- Funciona sem infra extra.
- Permite buscar logs relacionados a mesma operacao.

Desvantagens:
- Precisa disciplina para propagar o header.
- Logs ainda saem pelo stdout.

Alternativa B: log centralizado custom
- Um servico extra recebe todos os logs.
- Isso adiciona complexidade e nao e necessario aqui.

Alternativa C: console.log simples
- Facil de implementar, mas nao faz correlacao entre servicos.

Alternativa D: usar biblioteca de logging como Winston ou Pino
- Poderoso, mas traz dependencias a mais e aumenta a curva de aprendizado.

3. Decisao

Usar logging estruturado com Correlation IDs.

4. Consequencias

Positivas:
- Requisicoes podem ser rastreadas entre servicos.
- Debug fica mais rapido.
- O projeto ja fica pronto para observabilidade basica.

Negativas:
- Pequeno overhead de gerar o UUID.
- E necessario propagar o header entre servicos.

5. Implementacao

No Express, middleware faz:
- ler X-Correlation-ID no request ou gerar um UUID novo.
- guardar em req.correlationId.
- setar X-Correlation-ID na resposta.
- registrar um log JSON com service, correlationId, method, url e timestamp.

Quando um servico chama outro, ele inclui o header X-Correlation-ID na requisicao.

6. Validacao e testes

Testes feitos:
- O correlation id e gerado quando nao existe.
- O header e propagado entre servicos.
- Os logs aparecem em JSON.
- O mesmo correlation id pode ser encontrado em varios servicos.
