ADR-005: USAR INTERFACE WEB ESTATICA SERVIDA POR EXPRESS

Status: ACEITO
Data: 24/04/2026
Autor: Equipe de Desenvolvimento

1. Contexto

Queremos uma interface de usuario simples para criar e ver demandas, forcar verificacoes e ver logs.
A ideia e manter a interface leve e facil de manter.

2. Alternativas consideradas

Alternativa A: HTML/CSS/JS estatico servido por Express - escolhido
- Usa arquivos estaticos simples.
- Nao precisa de framework pesado.
- So precisa de Express para servir os arquivos.

Alternativa B: React, Vue ou Angular
- Muito mais trabalho e depende de build.
- Overkill para uma interface simples.

Alternativa C: motor de template como EJS ou Pug
- Poderia funcionar, mas mistura frontend e backend.
- Menos interatividade do lado do cliente.

Alternativa D: framework CSS como Bootstrap ou Tailwind
- Da estilo rapido, mas adiciona dependencias.
- Nao e necessario para este projeto.

3. Decisao

Usar HTML/CSS/JS estatico servido por Express.

4. Consequencias

Positivas:
- Rapido de desenvolver.
- Leve e facil de manter.
- Funciona em qualquer navegador.

Negativas:
- Menos recursos prontos do que um framework moderno.
- Sem hot reload automatico.

5. Implementacao

O UI Service tem:
- index.js como servidor Express.
- public/index.html para a pagina.
- public/app.js para a logica do cliente.

O frontend faz chamadas fetch para os outros servicos:
- GET /demands para listar demandas
- POST /demands para criar demandas
- POST /check-deadlines para forcar a verificacao
- GET /logs para ver historico
